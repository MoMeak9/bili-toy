# 本地音频实验室 PRD & Spec 补充版 v0.3

## 新增要求：单页应用，开箱即用

---

# 1. 新增产品原则

## 1.1 单页应用

产品采用 **Single Page Application，SPA** 形态。

所有核心能力集中在一个 Web 页面中完成：

```text
首页入口
↓
上传 / 录音 / 示例
↓
编辑器状态
↓
预设 / 效果 / 导出
↓
下载文件
```

页面不跳转到独立工作台，不进入多级页面，不依赖后端页面路由。

---

## 1.2 开箱即用

用户进入页面后，应立即看到核心操作：

```text
上传音频
开始录音
打开示例
```

用户无需：

* 登录
* 注册
* 创建项目
* 选择套餐
* 阅读复杂教程
* 配置音频设备
* 安装插件
* 连接网络接口
* 开通权限后才能浏览功能

除浏览器麦克风录音必须授权外，其余功能都应尽量直接可用。

---

# 2. 产品定位更新

## 2.1 新定位

**本地音频实验室是一款基于 Tone.js 的单页本地音频修改工具。**

用户打开网页后即可上传音频、开始录音、套用变声预设、调整专业效果并导出音频。

产品不使用 AI，不上传音频，不依赖后端接口，不要求账号登录，适合快速处理声音和进行趣味音频实验。

---

## 2.2 新产品一句话

```text
打开网页，即刻在浏览器本地剪辑、变声和导出你的声音。
```

---

## 2.3 新产品关键词

```text
单页应用
开箱即用
Tone.js 驱动
本地处理
无 AI
无上传
无接口
支持移动端
支持快捷键
有趣变声
专业导出
```

# 4. 单页应用用户流程

## 4.1 首次进入

```text
用户打开页面
↓
显示首页态 EmptyLanding
↓
用户点击上传音频 / 开始录音 / 打开示例
↓
触发 Tone.start()
↓
加载音频
↓
同一页面切换为编辑器态
```

---

## 4.2 上传音频

```text
点击上传音频
↓
打开本地文件选择器
↓
读取文件
↓
解析音频
↓
生成波形
↓
进入编辑器态
```

不得跳转页面。

---

## 4.3 打开示例

```text
点击打开示例
↓
加载内置示例音频
↓
初始化 Tone.js
↓
进入编辑器态
↓
默认展示预设面板
```

示例音频用于实现真正的开箱即用。用户即使没有准备音频，也能立即体验变声效果。

---

## 4.4 开始录音

```text
点击开始录音
↓
请求麦克风权限
↓
进入录音状态
↓
录音结束
↓
生成音频 Buffer
↓
进入编辑器态
```

仍在同一 SPA 页面内完成。

# 5. 开箱即用体验要求

## 5.1 首页首屏必须包含

| 元素     | 说明                  |
| ------ | ------------------- |
| 产品名    | 本地音频实验室             |
| 一句话价值  | 打开网页，即刻本地剪辑和改造声音    |
| 上传音频   | 主按钮                 |
| 开始录音   | 次按钮                 |
| 打开示例   | 次按钮                 |
| 本地隐私说明 | 文件不会上传              |
| 趣味预设入口 | 展示机器人、魔鬼低音、松鼠音等     |
| 专业能力入口 | 展示 Tone.js 效果链、导出参数 |

---

## 5.2 示例音频要求

为了支持开箱即用，应内置至少 1 条示例音频。

### 示例音频要求

| 项目 | 要求                  |
| -- | ------------------- |
| 时长 | 5–15 秒              |
| 内容 | 无版权风险，可使用自录语音或合成提示音 |
| 格式 | WAV 或 MP3           |
| 体积 | 小于 500KB            |
| 存放 | 应用静态资源目录            |
| 作用 | 让用户立即试听变声预设         |

### 示例音频交互

用户点击“打开示例”后：

1. 同页进入编辑器。
2. 自动加载示例音频。
3. 不自动播放，避免浏览器限制。
4. 高亮“声音预设”区域。
5. 展示提示：

```text
试试点击「机器人」或「魔鬼低音」听听效果。
```

# 6. SPA 路由要求

## 6.1 路由策略

MVP 不使用多页面路由。

推荐只保留：

```text
/
```

可选使用 URL Hash 保存轻量状态：

```text
/#editor
/#recording
/#export
```

但这些 Hash 仅用于刷新恢复 UI 状态，不代表真实页面跳转。

---

## 6.2 不允许的路由设计

不建议：

```text
/home
/editor
/project/:id
/export
/settings
/login
/dashboard
```

原因：

1. 破坏开箱即用感。
2. 增加用户理解成本。
3. 容易引入账号、工程、云端等复杂产品方向。
4. 不符合纯工具定位。

---

# 7. 状态机 Spec

## 7.1 App 状态

```ts
type AppMode =
  | "landing"
  | "loading"
  | "editor"
  | "recording"
  | "exporting"
  | "error";
```

---

## 7.2 状态说明

| 状态        | 说明          |
| --------- | ----------- |
| landing   | 初始首页态，无音频   |
| loading   | 正在解析音频或生成波形 |
| editor    | 已加载音频，可编辑   |
| recording | 正在录音        |
| exporting | 正在本地渲染和导出   |
| error     | 出现错误，可恢复    |

---

## 7.3 状态切换

```text
landing
├── upload file → loading → editor
├── open sample → loading → editor
└── start recording → recording → loading → editor

editor
├── export → exporting → editor
├── new project → landing
├── open file → loading → editor
└── error → error → editor / landing
```

# 10. 开箱即用开发要求

项目不得要求：

* 配置数据库
* 配置登录服务

* 配置转码服务

允许内置：

* 图标
* 字体，优先系统字体

* 默认预设配置

MVP 不需要任何 `.env`。

如必须配置，也只能用于：

VITE_ENABLE_PWA

不得出现：

```text
API_BASE_URL
OPENAI_API_KEY
S3_BUCKET

```

# 12. 移动端开箱即用

## 12.1 移动端首页

移动端首屏不隐藏核心 CTA。

按钮顺序：

```text
上传音频
开始录音
打开示例
```

---

## 12.2 移动端编辑器

进入编辑器后，默认底部抽屉打开在“预设”标签。

原因：

1. 趣味性更强。
2. 用户更快听到变化。
3. 减少复杂参数带来的理解成本。

---

## 12.3 移动端不展示内容

移动端默认不展示：

* 完整左侧文件树
* 完整右侧 Inspector
* 复杂效果链排序
* 复杂快捷键说明

这些内容通过“更多”或“高级”入口折叠。

# 13. 性能要求补充

## 13.1 SPA 首次加载

| 指标          | 要求             |
| ----------- | -------------- |
| 首屏可见        | 2 秒内           |
| 主 CTA 可点击   | 3 秒内           |
| JS 初始包      | 尽量控制在合理范围      |
| Tone.js     | 可随主包或按需加载      |
| ffmpeg.wasm | 不进入首屏主包，导出时懒加载 |
| 示例音频        | 不阻塞首页渲染        |

---

## 13.2 懒加载策略

| 模块            | 加载策略               |
| ------------- | ------------------ |
| 首页 UI         | 首屏加载               |
| Tone.js       | 首次音频交互前后加载         |
| wavesurfer.js | 进入编辑器时加载           |
| ffmpeg.wasm   | 用户选择 MP3/OGG 导出时加载 |
| 频谱图           | 用户打开专业分析时加载        |
| 示例音频          | 点击打开示例时加载          |

# 14. 验收标准补充

## 14.1 单页应用验收

| 场景      | 验收标准                |
| ------- | ------------------- |
| 打开首页    | 不跳转即可看到上传、录音、示例入口   |
| 上传音频    | 同一页面切换到编辑器          |
| 打开示例    | 同一页面加载示例并进入编辑器      |
| 开始录音    | 同一页面进入录音状态          |
| 导出      | 弹窗或抽屉内完成，不跳转页面      |
| 查看快捷键   | Dialog 内完成，不跳转页面    |
| 新建项目    | 返回 landing 状态，不刷新页面 |
| 移动端切换功能 | 通过底部抽屉或 Tab 完成，不跳页  |

---

## 14.2 开箱即用验收

| 场景    | 验收标准                              |
| ----- | --------------------------------- |
| 无音频文件 | 用户可点击打开示例体验                       |
| 无账号   | 所有核心功能可用                          |
| 无网络   | PWA 缓存后可打开应用并处理本地文件               |
| 无配置   | `npm install && npm run dev` 后可运行 |
| 无后端   | DevTools Network 不出现音频上传请求        |
| 首次用户  | 3 个 CTA 清晰可见                      |
| 移动端用户 | 首屏可直接上传、录音、示例                     |
| 桌面端用户 | 可使用 Space 直接播放/暂停                 |
